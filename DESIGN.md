# DESIGN.md — ツール_筋トレ記録

## 概要・背景

iPhoneで使う筋トレ記録アプリ。**PWA（Webアプリ）方式**で完全無料で開発・公開する。
将来的にApp Store公開（Capacitorでラップ、Apple Developer 年間12,980円）へ発展可能な構成。

- 開発環境：Windows 11（ローカルでHTML確認 → GitHub Pagesで公開 → iPhone実機確認）
- 利用シーン：ジムで片手操作。オフラインでも記録できること。
- ユーザー確認済み方針：まずPWA → 将来App Store（2026-07-08 決定）

## 決定事項（重要）

| 項目 | 決定 | 理由 |
|---|---|---|
| 公開方法 | GitHub Pages（無料） | リポジトリはpublicになる |
| データ保存 | 端末内（localStorage）＋ Googleスプレッドシート二重保存 | Phase 4でGAS連携 |
| ローカルDB | localStorage（抽象化レイヤー js/db.js 経由） | 記録データは軽量（数年分でも数百KB）。file://でも動作し確実。将来IndexedDBへ移行可能な構造 |
| GAS URLの扱い | **コードに埋め込まない**。アプリ内設定画面で入力し端末内にのみ保存 | publicリポジトリに秘密情報を置かないため |
| グラフ | Chart.js（MIT・無料）を予定（Phase 3） | |
| 推定1RM | Epley式：1RM = 重量 × (1 + 回数÷30) | |
| ボリューム定義 | Σ(重量 × 回数) | 部位別・期間別レポートの基礎数値 |

- [x] **Phase 1.4**：RM計算機（2026-07-09 実装 / v0.1.4）
  - 下部タブバーに「計算機」タブ追加（記録・履歴・グラフ・タイマー・計算機・設定 の6タブ）
  - 重量：**左右ドラッグのダイヤル式**（目盛りルーラー、0.1kg刻み、0〜300kg、中央ポインタ）
  - レップス：−/＋ボタンで増減（1〜30）
  - 1RM換算表：1〜12 Reps の重量をリアルタイム表示。1〜3=筋力アップ、4〜12=筋肥大のカテゴリ表示。現在のレップス行をハイライト
  - 換算式：1RM = 重量 × (1 + 0.0244 × (回数 − 1))（参考アプリの計算値と一致することをテストで確認済み）
  - 実装：ダイヤルは横スクロール要素のscrollLeft→重量変換（iOSの慣性スクロールがそのまま効く）
- [x] **Phase 1.5**：重量ドラムロールピッカー（2026-07-09 実装 / v0.1.5）
  - 記録画面の重量セルを、数字タップで開く**ドラムロール（ホイールピッカー）**式に変更
  - 上=軽い／下=重いの並び（0〜300kg・0.5kg刻み）。中央の選択値を黄色ハイライト＋kg単位表示
  - タッチはネイティブの慣性スクロール＋スクロールスナップ、マウスは上下ドラッグ／ホイールに対応
  - 「完了」で確定・「キャンセル」/背景タップで据え置き。±ボタン（2.5kg刻み）も従来どおり併存
  - 実装：`.drum-scroll`（scroll-snap-type: y mandatory）＋ scrollTop→index変換。項目高さ44pxはCSSと定数`DRUM_ITEM_H`で一致必須
- [x] **Phase 1.6**：種目の並べ替え（2026-07-09 実装 / v0.1.6）
  - 種目選択シートの各行に三本線グリップ（≡）を追加
  - グリップを**長押し**すると並べ替えモードに入り、ドラッグで順序を入れ替え可能（同じ器具グループ内で移動）
  - 「完了」ボタンで並べ替えモード終了。並べ替え結果は `state.exercises` に保存され永続化
  - 実装：`DB.reorderWithinPart(part, orderedIds)`で該当部位のみ再配置（他部位は保持）。ドラッグは pointer events + `elementFromPoint` によるライブ入れ替え
- [x] **Phase 1.7**：有酸素の記録項目対応（2026-07-10 実装 / v0.1.8）
  - 有酸素（部位=有酸素）の種目カードを、重量kg×回数ではなく**6項目**（時間(分)・距離(km)・速度(km/h)・傾斜(%)・カロリー(kcal)・心拍(bpm)）の2列グリッド入力に切り替え
  - 切り替え判定は記録カードに保存済みの `part === '有酸素'`（種目マスタ削除後も履歴が壊れない現行方式を踏襲）
  - 有酸素は「＋ セッション追加」で複数行を記録可能（インターバル対応）。新規追加時の既定行数は1（筋トレは3のまま）
  - 有酸素カードでは kg/回数/ドラムロール/±ステッパー/VOL(kg)を出さず、フットは「計 ○分 · ○km」の集計表示
  - 前回参考表示・履歴明細も有酸素向け（時間/距離/カロリー）に整形
  - 日次サマリータイルを内容のある種類だけ表示（筋トレ=レップ/負荷量、有酸素=時間/距離）。両方ある日は自動折り返し
  - CSVに有酸素6項目の列を追加（`時間min,距離km,速度kmh,傾斜%,カロリーkcal,心拍bpm`）
  - **注意**：本改修より前に有酸素種目を「kg×回数」で記録していた場合、その日は新レイアウトになり旧値は非表示（データ自体は残存）
- [x] **Phase 1.7.1**：有酸素グリッドのはみ出し修正（2026-07-10 / v0.1.9）
  - 有酸素の2列グリッドが `grid-template-columns: 1fr 1fr` だと `1fr(=minmax(auto,1fr))` の auto 最小値が数値入力の中身に引きずられ、右カラム（距離/傾斜/心拍）がカード外へはみ出していた
  - `minmax(0, 1fr)` ＋ `.cf { min-width: 0 }` でカラムを確実に縮小させて解消（Chromeヘッドレスで実測・スクショ確認済み。360px幅で内部オーバーフロー0）
- [x] **Phase 2（タイマー）**：インターバルタイマー（2026-07-10 実装 / v0.2.0）
  - プリセット 1分/3分/5分（タップで即スタート）＋カスタムホイール（1分刻み・最大15分、重量ドラムと同じ上下スクロール操作）。カスタム値はlocalStorage(`kintore_timer_min`)に保存
  - カウントダウンは**終了時刻の実時刻(endAt)ベース** → バックグラウンド復帰時に経過を自己補正（iOSでJSが止まっても復帰時に正しく反映／終了済みなら終了処理）
  - 円形リング表示、一時停止/再開・＋30秒・リセット。終了後は**1回で停止**（「もう一度」で同じ時間を再スタート）
  - 終了時の知らせ（v0.2.1）：終了表示「**TIME UP**」＋ビープ音**約5秒**（Web Audio・0.5秒間隔で11回・タップ時にアンロック）＋OS通知（許可時・iOS16.4+のPWA）＋アプリバッジ（`setAppBadge`）＋バイブ約5秒（`navigator.vibrate`・**対応端末のみ／iPhoneのSafari・PWAは非対応で無効**）＋作動中は画面スリープ抑止（Wake Lock）。リセット/再スタートで音・バイブを停止
  - **iOS制約（既知）**：画面ロック中やアプリ終了中はJS/タイマーが停止するため、ロック中の確実な通知発火はWeb単体では不可（サーバー経由のWeb Pushが必要＝将来課題）。「アプリを開いている間」は確実に作動

## データモデル（localStorage キー: `kintore_v1`）

```js
{
  version: 2,
  exercises: [ { id, name, part, equip,       // part: 胸/背中/脚/肩/腕/腹/有酸素/その他
                 video, note } ],             // equip: バーベル/ダンベル/マシン/ケーブル/自重/''（なし）
                                              // video: 参考動画URL（任意）/ note: フォームメモ（任意）
  workouts: {
    "2026-07-08": {
      date: "2026-07-08",
      condition: 5,                            // 1〜5（null=未設定）
      memo: "調子よし",
      entries: [
        { id, exId, name, part, equip,         // name/part/equip は記録時点の値を保持（種目削除後も履歴が壊れない）
          sets: [ { w: 60, r: 10 } ] }         // 筋トレ: { w:重量kg, r:回数 }
                                               // 有酸素(part==='有酸素'): { t:分, d:km, sp:km/h, inc:%, cal:kcal, hr:bpm }
      ]
    }
  }
}
```

- マイグレーション：`db.js` の `migrate()` が version を見て自動で引き上げる（v1→v2: equip付与・有酸素種目追加）
- CSV列：`日付,曜日,部位,種目,器具,セット,重量kg,回数,ボリュームkg,時間min,距離km,速度kmh,傾斜%,カロリーkcal,心拍bpm,メモ`（筋トレ行は有酸素6列が空、有酸素行は重量/回数/ボリュームが空）

## UIデザイン

- テーマ：ダーク（ジム・鉄の質感）＋ボルトイエロー `#d7ff3e` アクセント
- 数字フォント：Bebas Neue（Google Fonts・オフライン時はシステムフォントにフォールバック）
- 本文：日本語システムフォント（Hiragino / Yu Gothic）
- 下部タブバー：記録・履歴・グラフ・タイマー・計算機・設定（タイマーはPhase 2で実装済み、グラフはPhase 3まで準備中表示）
- 入力：±ステッパー（重量±2.5kg、回数±1）＋直接入力（数値キーボード）

## 実装ノート・制約

- **iOSのPWA制約**：画面ロック中のタイマー通知音は制約あり → Phase 2でWake Lock（画面スリープ防止）で対策予定
- **Safariのストレージ削除リスク**：ホーム画面追加したPWAは比較的永続だが、保険としてCSVエクスポート＋Phase 4のスプシ同期で二重化
- CSV仕様：`日付,曜日,部位,種目,セット,重量kg,回数,ボリュームkg,コンディション,メモ`（BOM付きUTF-8・CRLF）
- Service Worker登録は `location.protocol` がhttpの時のみ（file://でのローカル確認時はスキップ）
- ローカル確認方法：`index.html` をダブルクリックしてブラウザで開く（データはそのブラウザ内に保存される）
